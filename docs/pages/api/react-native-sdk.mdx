# Authenta React Native SDK

The Authenta React Native SDK enables seamless integration of Authenta's AI-powered media analysis and deepfake detection capabilities into your React Native applications.

With the SDK, you can:

- Capture images and videos directly from the device camera
- Upload media securely to the Authenta platform
- Run AI detection and verification workflows
- Poll and retrieve analysis results automatically
- Integrate liveness verification and identity workflows

**GitHub Repository:** https://github.com/phospheneai/authenta-reactnative-sdk

---

## Getting Started

### Installation

Install the SDK:

```bash
npm install @authenta/react-native-sdk
```

Install the required peer dependencies:

```bash
npm install react-native-vision-camera react-native-image-picker
```

---

## Authentication & Initialization

Create an `AuthentaClient` using your Authenta API token.

```typescript
import { AuthentaClient } from '@authenta/react-native-sdk';

const client = new AuthentaClient({
  api: 'Bearer YOUR_API_TOKEN',
  baseUrl: 'https://platform.authenta.ai',
});
```

### Configuration

| Property | Required | Description |
|----------|----------|-------------|
| `api` | Yes | Bearer token used to authenticate API requests |
| `baseUrl` | No | Authenta API endpoint. Defaults to `https://platform.authenta.ai` |

## Quick Start

The SDK provides the `AuthentaCapture` component which handles:

- Camera capture
- Upload workflow
- Processing
- Result polling
- Error handling

```typescript
import React, { useState } from 'react';
import { Button } from 'react-native';
import {
  AuthentaClient,
  AuthentaCapture
} from '@authenta/react-native-sdk';

const client = new AuthentaClient({
  api: 'Bearer YOUR_API_TOKEN',
});

export default function App() {
  const [visible, setVisible] = useState(false);

  return (
    <>
      <Button
        title="Start Verification"
        onPress={() => setVisible(true)}
      />

      <AuthentaCapture
        client={client}
        visible={visible}
        onClose={() => setVisible(false)}
        onResult={(result) => {
          setVisible(false);
          console.log(result);
        }}
        onError={(error) => {
          setVisible(false);
          console.error(error);
        }}
        livenessCheck={true}
      />
    </>
  );
}
```

## Processing Flow

The SDK automatically manages the complete Authenta processing lifecycle:

```text
Capture Media
      ↓
Create Upload Job
      ↓
Upload Media
      ↓
Finalize Upload
      ↓
Start Analysis
      ↓
Poll Status
      ↓
Receive Results
```

No manual handling of `jobId`, uploads, or result polling is required when using `AuthentaCapture`.

## Error Handling

Provide an `onError` callback to handle SDK and API errors.

```typescript
onError={(error) => {
  console.error(error);
}}
```
